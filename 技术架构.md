## 整体架构：分层设计

```
┌─────────────────────────────────────────────────┐
│                  Presentation Layer               │  React + TailwindCSS + Framer Motion
│                  (UI组件层)                        │
├─────────────────────────────────────────────────┤
│                  View Model Layer                 │  Zustand Stores（UI状态的读写层）
│                  (视图模型层)                      │
├─────────────────────────────────────────────────┤
│                  Game Engine Layer                │  纯TypeScript，不依赖React
│                  (游戏引擎层)                      │
│  ┌──────────┐ ┌──────────┐ ┌──────────────────┐ │
│  │ Market   │ │ AI       │ │ Event            │ │
│  │ Engine   │ │ Engine   │ │ Engine           │ │
│  ├──────────┤ ├──────────┤ ├──────────────────┤ │
│  │ Sentiment│ │ News     │ │ Country          │ │
│  │ Engine   │ │ Engine   │ │ Engine           │ │
│  └──────────┘ └──────────┘ └──────────────────┘ │
├─────────────────────────────────────────────────┤
│                  Data Layer                      │  游戏数据、配置、存档
│                  (数据层)                         │
└─────────────────────────────────────────────────┘
```

**核心原则**：
1. 游戏引擎层完全独立于UI，不引用任何React代码，可独立运行和测试
2. UI层通过Zustand读取引擎状态来渲染
3. 引擎之间通过EventBus松耦合通信
4. 配置数据与引擎逻辑完全分离

## GameLoop 核心实现

```typescript
// engine/core/GameLoop.ts

class GameLoop {
  private tickAccumulators = {
    L0: 0, L1: 0, L2: 0, L3: 0, L4: 0, L5: 0
  }
  private tickIntervals = {
    L0: 500,   // 0.5秒 - 价格更新
    L1: 2000,  // 2秒   - AI决策
    L2: 3000,  // 3秒   - 情绪更新
    L3: 5000,  // 5秒   - 事件检查
    L4: 30000, // 30秒  - 宏观更新
    L5: 90000  // 60-90秒 - 重大事件
  }

  private lastTime = 0
  private paused = false

  start() {
    this.lastTime = performance.now()
    const loop = (now: number) => {
      if (!this.paused) {
        const delta = now - this.lastTime
        this.lastTime = now
        this.update(delta)
      }
      requestAnimationFrame(loop)
    }
    requestAnimationFrame(loop)
  }

  update(deltaMs: number) {
    for (const level of ['L0','L1','L2','L3','L4','L5'] as const) {
      this.tickAccumulators[level] += deltaMs
      if (this.tickAccumulators[level] >= this.tickIntervals[level]) {
        this.tickAccumulators[level] -= this.tickIntervals[level]
        this.executeTick(level)
      }
    }
    // AI分帧执行（每帧处理一小批）
    this.aiEngine.frameUpdate()
  }

  private executeTick(level: string) {
    switch(level) {
      case 'L0': this.marketEngine.tickPrices(); break
      case 'L1': this.aiEngine.scheduleTick(); break
      case 'L2': this.sentimentEngine.tick(); break
      case 'L3': this.eventEngine.tick(); break
      case 'L4': this.countryEngine.tick(); break
      case 'L5': this.eventEngine.checkBlackSwan(); break
    }
  }
}
```

## EventBus 设计

```typescript
// engine/core/EventBus.ts

type EventMap = {
  'price:updated':       { assetId: string; price: number; change: number }
  'sentiment:changed':   { level: 'global'|'country'|'asset'; target: string; value: number; delta: number }
  'ai:trade':            { aiId: string; aiType: string; assetId: string; side: 'buy'|'sell'; amount: number }
  'ai:kol:post':         { aiId: string; content: string; targetAsset: string }
  'news:published':      { newsId: string; template: string; credibility: number; spread: number }
  'news:debunked':       { newsId: string; source: string }
  'event:triggered':     { eventId: string; eventType: string; impact: Record<string, number> }
  'war:started':         { attacker: string; defender: string }
  'war:update':          { attacker: string; defender: string; result: string }
  'war:ended':           { winner: string; loser: string }
  'player:trade':        { assetId: string; side: 'buy'|'sell'; amount: number; leverage: number }
  'player:kol:action':   { action: string; target: string; effect: number }
  'player:news:publish': { newsId: string; template: string }
  'player:rugpull':      { tokenId: string; profit: number }
  'country:policy':      { countryId: string; policy: string; effect: Record<string, number> }
  'crash:triggered':     {}
  'circuit:breaker':     { duration: number }
}

class EventBus {
  private listeners = new Map<string, Set<Function>>()

  on<K extends keyof EventMap>(event: K, fn: (data: EventMap[K]) => void) {
    if (!this.listeners.has(event)) this.listeners.set(event, new Set())
    this.listeners.get(event)!.add(fn)
    return () => this.listeners.get(event)?.delete(fn) // 返回取消订阅函数
  }

  emit<K extends keyof EventMap>(event: K, data: EventMap[K]) {
    this.listeners.get(event)?.forEach(fn => fn(data))
  }
}
```

## AI 分帧执行 + Worker 预留

```typescript
// engine/ai/AIEngine.ts

class AIEngine {
  private pendingQueue: AIAgent[] = []
  private bufferedTrades: AITradeDecision[] = []
  private batchSize = 5  // 每帧处理5个AI

  scheduleTick() {
    this.pendingQueue = this.pool.getActiveAgents()
  }

  frameUpdate() {
    if (this.pendingQueue.length === 0) return

    const batch = this.pendingQueue.splice(0, this.batchSize)
    for (const agent of batch) {
      const decision = agent.decide(this.marketSnapshot, this.sentimentSnapshot)
      if (decision) {
        this.bufferedTrades.push(decision)
      }
    }

    if (this.pendingQueue.length === 0) {
      this.flushTrades()
    }
  }

  private flushTrades() {
    for (const trade of this.bufferedTrades) {
      this.eventBus.emit('ai:trade', trade)
    }
    this.bufferedTrades = []
  }
}
```

```typescript
interface AIComputeBridge {
  computeDecisions(agents: AIAgentData[], market: MarketSnapshot): Promise<AITradeDecision[]>
}

// 主线程版本（默认）
class MainThreadBridge implements AIComputeBridge {
  computeDecisions(agents, market) {
    return Promise.resolve(this.runLocal(agents, market))
  }
}

// Worker版本（预留，后续切换用）
class WorkerBridge implements AIComputeBridge {
  private worker: Worker
  computeDecisions(agents, market) {
    return new Promise(resolve => {
      this.worker.postMessage({ agents, market })
      this.worker.onmessage = (e) => resolve(e.data)
    })
  }
}

// 使用时只需切换实现
const bridge: AIComputeBridge = new MainThreadBridge()  // 后续改为 new WorkerBridge()
```

```typescript
// engine/meta/SaveSystem.ts

interface SaveData {
  version: number
  timestamp: number
  seed: number
  tick: number

  player: PlayerSnapshot
  market: MarketSnapshot
  sentiment: SentimentSnapshot
  countries: CountrySnapshot[]
  aiAgents: AIAgentSnapshot[]
  newsHistory: NewsRecord[]
  eventHistory: EventRecord[]

  metaPoints: number
  unlockedCharacters: string[]
  unlockedTemplates: string[]
  achievements: string[]
}

class SaveSystem {
  private autoSaveInterval = 30000  // 每30秒自动存档

  snapshot(): SaveData {
    return {
      version: 1,
      timestamp: Date.now(),
      seed: this.runSeed,
      tick: this.gameLoop.currentTick,
      player: this.playerState.serialize(),
      market: this.marketEngine.serialize(),
      sentiment: this.sentimentEngine.serialize(),
      countries: this.countryEngine.serialize(),
      aiAgents: this.aiEngine.serialize(),
      newsHistory: this.newsEngine.serialize(),
      eventHistory: this.eventEngine.serialize(),
      metaPoints: this.metaProgression.points,
      unlockedCharacters: this.metaProgression.unlockedCharacters,
      unlockedTemplates: this.metaProgression.unlockedTemplates,
      achievements: this.metaProgression.achievements,
    }
  }

  restore(data: SaveData) {
    this.playerState.deserialize(data.player)
    this.marketEngine.deserialize(data.market)
    this.sentimentEngine.deserialize(data.sentiment)
    this.countryEngine.deserialize(data.countries)
    this.aiEngine.deserialize(data.aiAgents)
    this.newsEngine.deserialize(data.newsHistory)
    this.eventEngine.deserialize(data.eventHistory)
  }

  async save(storage: StorageBackend) {
    const data = this.snapshot()
    await storage.write('save.json', JSON.stringify(data))
  }

  async load(storage: StorageBackend): Promise<SaveData> {
    const raw = await storage.read('save.json')
    return JSON.parse(raw)
  }
```

### 存档边缘情况处理

**存档时机限制：**

| 场景 | 能否保存 | 说明 |
|------|---------|------|
| 正常游戏中 | 随时保存 | |
| 暂停状态 | 可保存 | 最佳保存时机 |
| 战争进行中 | 可保存 | 恢复时从当前战争阶段继续 |
| KOL对战进行中 | 不可保存 | 对战仅30秒，等打完再存 |
| 事件链执行中 | 不可保存 | 事件动画/弹窗播放完才可存 |
| 结算界面 | 自动保存 | 无需手动 |
| 事件编辑器打开时 | 不可保存 | 关闭编辑器后可存 |
| 交易确认弹窗 | 不可保存 | 交易完成/取消后可存 |

**SaveData扩展字段：**

```typescript
interface SaveData {
  // ... 原有字段 ...

  // 战争状态（用于战争进行中存档）
  activeWar?: {
    phase: WarPhase
    participants: string[]
    scores: Record<string, number>
    elapsed: number
    economyDamage: Record<string, number>
  }

  // 冷却状态
  cooldowns: Record<string, number>  // 操作ID → 剩余tick

  // 假新闻传播状态
  activeFakeNews: Array<{
    id: string
    templateId: string
    targetCountry: string
    credibility: number
    age: number  // 已存活tick数
    channel: string
  }>
}

// 存档校验
class SaveValidator {
  validate(data: SaveData): { valid: boolean; error?: string } {
    if (data.version !== CURRENT_VERSION) {
      return { valid: true }  // 安全降级，不崩溃
    }
    // 校验关键字段
    if (!data.seed || !data.tick) {
      return { valid: false, error: '存档已损坏' }
    }
    return { valid: true }
  }
}
```

**自动保存规则：**

- 每隔5分钟自动保存一个快照（覆盖式）
- 重大事件触发时额外保存（战争爆发、黑天鹅、崩盘）
- 最多保留：1个自动存档 + 1个事件存档 = 2个自动槽位
- 手动存档槽位：3个（玩家自由命名）
- 沙盒模式：只保留1个手动存档槽位

## 性能优化策略

```
AI计算:
├── 分帧执行（每帧4-5个AI，避免卡顿）
├── 对象池（AIPool预创建，避免GC）
├── 活跃度分级（有持仓的每tick算，休眠的跳过）
└── 预留Web Worker接口（后续需要时切换）

K线图:
├── Lightweight Charts 原生高性能渲染
├── 只维护最近500根K线的渲染数据
└── 增量更新（只推新数据，不全量刷新）

状态更新:
├── Zustand 浅比较（避免不必要的重渲染）
├── 热路径数据用 ref 而非 state（避免触发渲染）
└── 防抖高频更新（AI日志等不需要每条都渲染）

内存:
├── 新闻/日志只保留最近200条
├── K线历史只保留最近500根
└── 休眠AI不持有详细状态
```

## 技术决策记录

| 决策 | 选择 | 理由 |
|------|------|------|
| AI计算位置 | 主线程，预留Worker接口 | 简单优先，后续按需迁移 |
| 存档方案 | JSON快照 | 简单可靠，每30秒自动存 |
| K线图渲染 | Lightweight Charts | TradingView官方，轻量金融级 |
| 状态管理 | Zustand | 轻量，适合高频更新，API简洁 |
| 引擎间通信 | EventBus发布订阅 | 松耦合，引擎可独立测试 |
| 确定性 | 可种子随机数 | 同种子=同结果，方便测试和复现 |
| 数据分离 | data/目录独立配置 | 改数值不改代码 |
